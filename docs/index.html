What it does

A multi-language obfuscation toolkit powered by rust-analyzer's semantic analysis. Designed for protecting proprietary code and stripping PII before sharing.

λ

Semantic Renaming

Uses rust-analyzer to rename identifiers across your entire codebase while respecting scope, visibility, and macro boundaries.

⬡

Control Flow Flattening

Transforms sequential code into state-machine loops, making reverse engineering significantly harder.

◇

Dead Code Injection

Inserts meaningless computations that execute but produce no side effects, cluttering static analysis.

⊘

Opaque Predicates

Wraps code blocks in always-true conditions that are difficult for decompilers to optimize away.

⌁

String Encryption

XOR-encodes string literals at compile time, decoded at runtime to hide sensitive text from binaries.

⊗

PII Stripping

Automatically redacts emails, local paths, and author information. Configures release builds to strip symbols.

Installation

Clone the repository and build with Cargo. Requires Rust 1.70+ with the 2021 edition.

# Clone the repository
git clone https://github.com/1-void/obfuscator.git
cd obfuscator

# Build release binary
cargo build --release

# Optional: install to PATH
cargo install --path .

// commands

Usage

obfuscator --input ./my-project --output ./obfuscated

CLI Options

FlagDescriptionDefault

Example: Full Obfuscation

obfuscator \
  --input ./src \
  --output ./dist \
  --mode aggressive \
  --strip-pii \
  --seed 0xDEADBEEF

Obfuscation Modes

Safe Mode

Renames private identifiers only. Preserves public API, comments, and formatting. Best for code sharing where compilation must be guaranteed.

  • ✓ Identifier renaming
  • ✓ PII stripping (if enabled)
  • ✗ Dead code injection
  • ✗ Control flow changes

Aggressive Mode

Maximum obfuscation. Applies all transformations including string encryption, control flow flattening, and opaque predicates.

  • ✓ Everything in Safe
  • ✓ String literal encryption
  • ✓ Dead code injection
  • ✓ Control flow flattening
  • ✓ Opaque predicates
  • ✓ HTML/JS/CSS minification

How It Works

Parse Source
→
Index Definitions
→
Plan Renames
→
Transform AST
→
Emit Code
  • rust-analyzer integration Uses ra_ap_ide for semantic analysis, ensuring renames are valid across module boundaries, trait impls, and re-exports.
  • syn / quote AST manipulation Parses Rust with syn and rewrites the token stream with quote, enabling fine-grained transformations like opaque predicate insertion.
  • Macro-aware rewriting Custom token stream visitor ensures identifiers inside macros (e.g., println!) are renamed consistently with their definitions.
  • Deterministic output Optional seed parameter guarantees reproducible obfuscation — same input + seed = same output.

// safety

What's Protected

The obfuscator is designed to preserve functionality. These items are never renamed:

Keywords & Builtins

fn, let, impl, Self, main, etc.

Public API

Items marked pub retain their names for external compatibility.

FFI Boundaries

Items with #[no_mangle], #[export_name], or #[link_name].
