// what it does

features

multi-layer obfuscation using rust-analyzer's semantic engine. protects proprietary code, strips PII, makes reverse engineering expensive.

λ

semantic renaming

uses rust-analyzer to track every identifier across modules, traits, macros. renames user_id to k9vmqxh everywhere it's referenced.

⬡

control flow flattening

converts functions into state machines. your for loop becomes a match statement with 12 states. decompilers hate this.

◇

dead code injection

inserts pointless calculations that execute but do nothing. let _ = (x * 17 + 42) % 3; clutters static analysis tools.

⊘

opaque predicates

wraps blocks in always-true conditions like if (23+19) == 42. compilers can't prove it, so they can't optimize it out.

⌁

string encryption

XORs string literals at compile time. "password" becomes byte array + runtime decode. strings invisible in binaries.

⊗

PII stripping

removes emails, paths, author names from Cargo.toml. enables strip = true in release profile. clean shareable builds.

# clone + build
git clone https://github.com/1-void/obfuscator.git
cd obfuscator
cargo build --release

# install to PATH (optional)
cargo install --path .

obfuscation modes

safe mode

only renames private identifiers. keeps public API intact. preserves comments + formatting. guaranteed to compile.

  • ✓ identifier renaming
  • ✓ PII stripping (optional)
  • ✗ dead code injection
  • ✗ control flow changes

aggressive mode

all transformations enabled. string encryption, control flow flattening, opaque predicates. max protection.

  • ✓ everything in safe
  • ✓ string encryption
  • ✓ dead code injection
  • ✓ control flow flattening
  • ✓ opaque predicates
  • ✓ minification (html/js/css)

how it works

  • rust-analyzer integration uses ra_ap_ide for semantic analysis. tracks every definition across modules, traits, impl blocks, re-exports. renames are semantically valid.
  • syn / quote AST manipulation parses with syn, transforms the token stream with quote. enables surgical edits like opaque predicate insertion without breaking syntax.
  • macro-aware rewriting custom visitor handles token streams inside macros. identifiers in println! get renamed consistently with their definitions.
  • deterministic output optional --seed guarantees reproducible builds. same input + seed = identical output. useful for CI/CD pipelines.

what's never touched

these items are always preserved to maintain functionality:

keywords + builtins

fn, let, impl, Self, main — anything the compiler needs.

public API

pub items keep their names for external compatibility. callers won't break.

FFI boundaries

#[no_mangle], #[export_name], #[link_name] — C interop stays intact.
